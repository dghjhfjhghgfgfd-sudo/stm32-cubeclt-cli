# 浙江专科赛道_培训网资料蒸馏库

更新时间：2026-06-07

用途：把培训网已登录后可见的课程、回放、资料主题压缩为比赛当天可直接调用的判断库。本文不保留链接清单，只保留结论、解法路线、器件准备和赛题匹配规则。

适用对象：浙江省电子设计竞赛专科/高职赛道，兼顾全国赛高职组可迁移题型。

## 1. 总判断

浙江专科赛道最值得押注的不是单一高难方向，而是“嵌入式控制 + 传感测量 + 执行机构 + 基础模拟/电源”的组合题。比赛题通常不会只考一个芯片或一个算法，而是考能不能在有限时间内把系统做稳、测准、调通、展示清楚。

优先准备顺序：

1. 自动控制类：小车、二维平台、瞄准、跟踪、巡检、机械执行机构。
2. 测量仪器类：信号检测、元件/线路参数测量、简易测试仪、数据展示。
3. 视觉/非接触感知类：单目视觉、颜色/目标识别、超声/红外/激光测距、声光定位。
4. 综合嵌入式工程类：多传感器融合、通信、显示、人机交互、数据记录。
5. 模拟与电源基础类：运放、滤波、功率测量、D 类功放、小功率 DC-DC。
6. 高功率电力电子类：只作为备选，不建议专科队伍把主战场押在有源滤波、变流器、能量回馈这类高风险硬件上。

比赛当天优先策略：先做一个可测、可演示、可稳定复现的 60-70 分系统，再补精度、速度、自动化和附加功能。

## 2. 近期高概率题型

### A. 自动小车 / 移动执行装置

可能形式：

- 循迹、避障、定位、搬运、巡检、路径规划。
- 小车搭载摄像头、超声波、激光测距、编码器或 IMU。
- 场地内目标识别、坐标测量、自动靠近、自动返回。

推荐架构：

- 主控：STM32 / MSPM0 / ESP32，优先用团队最熟的。
- 底盘：两轮差速或四轮底盘，编码器电机优先。
- 感知：灰度/红外阵列 + 超声/TOF + 摄像头二选一或组合。
- 控制：速度闭环 PID + 路径状态机，不一开始追求复杂算法。
- 显示：OLED/LCD 显示模式、坐标、速度、误差。

核心得分点：

- 稳定走完全程比极限速度更重要。
- 转弯、停止、目标识别后的动作要可重复。
- 编码器闭环、动作状态机、异常超时保护是关键。

常见坑：

- 只靠延时控制电机，场地一变就失败。
- 摄像头识别没有光照兜底。
- 底盘机械松动、轮胎打滑、供电压降导致调试结果不稳定。

比赛当天首选方案：

1. 先用编码器闭环跑基本动作。
2. 再接入传感器做状态切换。
3. 最后补视觉或高级路径。

### B. 简易测量仪器 / 信号检测

可能形式：

- 线路故障检测、双绞线测试、元件参数测量。
- 信号频率、幅度、相位、占空比、阻抗、传输质量测量。
- 小型示波/频谱/扫频/数据记录装置。

推荐架构：

- 模拟前端：保护电阻、钳位、分压、运放缓冲、滤波。
- 采样：MCU ADC / 外置 ADC / 比较器捕获。
- 计算：频率用定时器捕获，幅度用 ADC，阻抗用已知激励 + 响应估算。
- 输出：LCD/OLED + 串口日志，必要时加蜂鸣器或状态灯。

核心得分点：

- 测量范围、误差、重复性、自动判别。
- 输入保护和校准流程。
- 展示界面清晰，比单纯串口打印更适合现场评分。

常见坑：

- 没有输入保护，误接后前端或 MCU 损坏。
- ADC 参考电压不稳，导致读数漂移。
- 用软件延时测频，精度不够。
- 忽略地线、屏蔽和接触电阻。

比赛当天首选方案：

1. 先确定被测量范围和评分精度。
2. 用最简单可校准模型实现基本测量。
3. 加入零点/满量程校准和多次平均。
4. 最后优化自动识别和界面。

### C. 视觉识别 / 自动瞄准 / 目标测量

可能形式：

- 单目视觉测距、目标定位、颜色/形状识别。
- 激光点、靶心、物体边界、运动目标跟踪。
- 云台、舵机、二维平台、发射/指示装置。

推荐架构：

- 视觉：OpenMV / ESP32-CAM / USB 摄像头 + 上位机，比赛中优先选成熟方案。
- 执行：二自由度舵机云台或步进电机平台。
- 控制：视觉误差 -> 舵机角度 PID/比例控制 -> 到达阈值停止。
- 校准：摄像头畸变、像素到角度/距离映射、目标颜色阈值。

核心得分点：

- 识别稳定、响应速度、目标居中误差。
- 光照变化下仍能工作。
- 手动/自动模式切换，便于现场兜底。

常见坑：

- 只在一种光照下调好阈值。
- 舵机抖动，没有死区和滤波。
- 没有机械限位，启动时乱转。
- 视觉坐标和执行方向符号搞反。

比赛当天首选方案：

1. 先做颜色/形状阈值识别，不先追复杂 AI。
2. 建立像素误差到舵机增量的闭环。
3. 加死区、限幅、低通滤波。
4. 再做距离估算、目标分类或复杂扩展。

### D. 通信 / 接收机 / 无线传输

可能形式：

- 简易接收机、无线传输信号模拟、调制解调、通信质量评估。
- 音频/低频信号传输、ASK/FSK/OOK、串口无线模块。

推荐架构：

- 如果题目允许模块化，优先用成熟无线模块或射频模块。
- 若要求自制接收链路，先保证前端放大、滤波、比较整形、解码。
- 数字处理用定时器捕获、Goertzel/FFT 或阈值判决。

核心得分点：

- 接收灵敏度、误码率、抗干扰、距离。
- 频率稳定和滤波。
- 可视化显示通信状态。

常见坑：

- 前端增益过高导致饱和。
- 没有带通滤波，噪声进入判决。
- 电源纹波影响接收。
- 调试只看串口，不看示波器波形。

比赛当天首选方案：

1. 先让链路能传固定码。
2. 再做同步头、校验和重发。
3. 最后优化滤波、距离和抗干扰。

### E. 模拟 / 运放 / 滤波 / 功率测量

可能形式：

- 运放参数测量、滤波器、有源滤波、音频功放、功率分析。
- 传感器前端调理、信号放大、峰值/有效值测量。

推荐架构：

- 运放优先选轨到轨、低失调、供电简单的器件。
- 滤波优先二阶有源滤波或 RC + 数字滤波组合。
- 功率测量用电压/电流采样 + RMS/平均功率计算。
- 音频类注意输入幅度、负载、电源和散热。

核心得分点：

- 指标可测、误差可解释、波形干净。
- PCB/接线布局和地线处理。
- 保护、散热、过流限幅。

常见坑：

- 运放供电范围不匹配。
- 输出摆幅达不到，误以为程序错。
- 面包板上做高频/功率电路，噪声和接触问题严重。
- 只仿真不实测。

比赛当天首选方案：

1. 先用成熟参考电路搭前端。
2. 示波器确认每一级波形。
3. 再接 MCU 采样。
4. 最后做校准和误差表。

### F. 无人机 / 复杂机器人

可能形式：

- 立体货架盘点、巡检、识别、定位。
- 对专科赛道来说，出现完整无人机题的概率低于小车/平台类，但视觉巡检思想可能迁移。

推荐策略：

- 如果题目是无人机，优先用成熟飞控，不自写底层飞控。
- 把得分重点放在识别、任务流程、通信、标记和安全降级。
- 若规则允许地面替代或简化平台，先做稳定可演示版本。

## 3. 赛题出现后的快速匹配规则

看到关键词后，按以下规则立即归类：

- 出现“路线、移动、巡检、搬运、定位、避障、靶区”：归自动小车/移动控制。
- 出现“频率、幅值、相位、阻抗、线路、参数、测试仪”：归测量仪器。
- 出现“图像、目标、颜色、形状、瞄准、单目、识别”：归视觉/瞄准。
- 出现“接收、发送、调制、解调、误码、无线”：归通信。
- 出现“功率、滤波、变流、逆变、能量、负载、D类”：归模拟/电源。
- 出现“综合测评、限定器件、现场抽测”：优先走最稳的最小系统，少做花活。

## 4. 比赛当天方案输出模板

收到赛题后，先按这个顺序给方案：

1. 赛题归类：判断属于哪一类或哪几类组合。
2. 指标拆解：基本要求、发挥要求、隐藏约束。
3. 最小可得分系统：4-6 小时内能搭出来的版本。
4. 推荐最终架构：主控、传感器、执行器、模拟前端、通信、显示。
5. 关键算法：PID、状态机、滤波、测量公式、校准方法。
6. 任务分工：硬件、软件、机械、测试报告。
7. 风险清单：最可能炸的点和替代方案。
8. 测试流程：用什么仪器、测哪些数据、如何证明指标。
9. 加分路线：在基本功能稳定后再加的内容。

## 5. 推荐通用硬件底座

主控优先级：

- 已熟练 STM32：继续用 STM32。
- 想贴近 TI 培训体系：MSPM0L1306 / MSPM0G 系列。
- 需要联网/摄像头/简单无线：ESP32。
- 不建议比赛当天临时换新平台。

必备模块：

- OLED/LCD 显示模块。
- 编码器电机、L298N/TB6612/DRV8833 或更可靠电机驱动。
- 舵机、步进电机驱动。
- 超声波/TOF/红外测距。
- 灰度传感器或循迹阵列。
- OpenMV 或成熟摄像头模块。
- 运放、比较器、基准源、常用阻容、电位器。
- 电流/电压采样模块。
- 稳压模块、锂电池、保险丝、开关、接插件。

调试仪器优先级：

- 万用表必须有。
- 示波器非常关键，尤其是测量、通信、模拟、电源题。
- 逻辑分析仪适合串口、PWM、编码器、通信调试。
- 可调电源能显著降低烧板风险。

## 6. 浙江专科队伍最优备战路线

不要平均用力。按“能现场做出来”的价值排序：

1. 练熟一个主控工程模板：GPIO、ADC、PWM、定时器捕获、串口、I2C/SPI、OLED。
2. 练熟一个小车/云台闭环模板：PID、限幅、状态机、急停。
3. 练熟一个测量前端模板：分压保护、运放缓冲、滤波、ADC 校准。
4. 练熟一个视觉模板：识别目标、输出坐标、控制舵机。
5. 练熟报告和测试表：现场数据、误差、照片、框图、流程图。

比赛当天最不该做的事：

- 从零学新芯片。
- 从零画复杂 PCB。
- 一开始追求发挥题，基本题还不稳定。
- 没有仪器验证，只靠“看起来能跑”。
- 代码没有模式切换和手动兜底。

## 7. 当前预测结论

若只押 3 个方向：

1. 小车/执行机构 + 传感器闭环。
2. 简易仪器/信号测量 + 显示校准。
3. 视觉识别/自动瞄准 + 舵机或二维平台。

若押 5 个方向：

1. 自动小车。
2. 测量仪器。
3. 视觉/非接触感知。
4. 运放/滤波/功率测量基础。
5. 简单通信/接收链路。

最像专科赛道的命题风格：题目表面是一个装置，实质是多个基础模块的工程集成。稳定性、调试能力和测试证明通常比算法名词更重要。

## 8. 我收到赛题后的工作方式

用户发来赛题原文、图片或 PDF 后，优先调用本文和本地题型库，直接输出：

- 最推荐路线。
- 备用路线。
- 不建议路线。
- 关键器件清单。
- 代码模块划分。
- 硬件连接和测试顺序。
- 评分项优先级。
- 当天 12/24/48/72 小时推进计划。

原则：给出能真实做出来、能调通、能拿分的方案，而不是只追求理论最优。
