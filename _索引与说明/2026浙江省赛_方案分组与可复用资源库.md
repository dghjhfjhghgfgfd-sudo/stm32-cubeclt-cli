## 2. 方案组 A：测量仪器 / 信号检测 / 参数估计

### 适配赛题

- 2024 B：单相功率分析仪
- 2025 D：简易以太网双绞线测试仪
- 2025 G：电路模型探究装置
- 2023 J：线路故障自动检测系统
- 2021 J：周期信号波形识别及参数测量装置
- 2022 参考：信号调制度测量、电子负载

### 为什么优先级最高

2026 浙江省赛很可能受 TI 杯模拟电子系统专题赛影响，测量、采集、运放、ADC、信号处理权重会变高。这个方向不依赖复杂机械，适合第一天做出基本功能。

### 基础方案

系统结构：

- 输入保护：限流电阻、钳位二极管、TVS 或运放输入保护。
- 模拟前端：分压、隔离、运放缓冲、二阶低通/带通滤波。
- 采样：MCU ADC + DMA，或外置 ADC。
- 数字测量：频率用定时器捕获，幅值/RMS 用 ADC，阻抗/模型用已知激励和响应估计。
- 显示：OLED/TFT 显示量程、数值、误差、状态。
- 校准：零点、满量程、比例系数、偏置补偿。

### 可复用模块

- ADC + DMA 连续采样模板。
- 定时器输入捕获测频模板。
- RMS/峰峰值/均值/方差计算。
- 移动平均 / 中值滤波 / 一阶低通。
- OLED 菜单和校准参数保存。
- 串口输出原始数据，便于调试。

第一天必须完成：

- 能采样。
- 能显示。

第二天再做：

- 自动量程。

### 风险

- 输入范围不清导致烧 ADC。
- 采样率不够导致波形/频率错误。
- 模拟前端没有共地/隔离处理。
- 没有校准流程，现场误差飘。

- 灰度循迹 PID。

### 新增补充（2024-2025 获奖/开源复盘蒸馏）

来源见本节末"参考来源链接"。以下只补本地基础方案未覆盖的内容。

共性（多份方案一致，可直接当默认路线）：

- 主控必须用 TI MSPM0 系列（2024 H 题硬性要求），首选 MSPM0G3507（80MHz）。比赛前必须提前把 MSPM0 底层驱动（PWM、定时器编码器、UART、ADC）准备好，否则现场移植驱动会爆肝。
- 电机驱动优先 TB6612（单芯片双路、最大 3A、支持高频 PWM），多份复盘明确不建议 L298N（PWM 频率上不去、发热严重）。靠左右轮 PWM 占空比差速转弯。
- 双环切换是主流控制结构：直道段用陀螺仪 Z 轴航向角做"角度环 PD"保持直线；有黑线段（弯道/半弧）用灰度阵列偏差量化做"循迹环 PD"。两种环用状态机按路段切换，PID 普遍只用 PD（去掉 I 防积分饱和），位置式实现。
- 声光提示用无阻塞（非 delay 阻塞）方式实现，避免在过点提示时打断控制循环——本地把"只靠延时控制"列为风险，这里给出正路。

差异 / 选型对比（按需取舍）：

- IMU 型号差异明显：维特 JY61P / JY901S 可串口直读角度、自带滤波，比 MPU6050 好用；MPU6050 零漂随温度漂移严重、误差大，多份复盘不推荐裸用。预算够上 HWT101（测航向专用，零漂更小）。部分扩展板直接集成 ICM20608。本地"IMU"建议具体落到 JY61P/JY901S 这一档。
- 陀螺仪零漂的两条弥补路线：① 上更好的航向传感器（HWT101）；② 用 MSPM0 定时器的编码器功能读双相做里程计，靠编码器距离纠正航向/定距，提高容错。
- 陀螺仪角度偏差必须做 ±180° 跨零点正确处理，否则掉头/大转角时角度环会突跳。
- 底盘：两轮 + 万向轮三轮即可完成 H 题，编码器非必须但强烈建议（用于定距与容错）；车体受限 ≤25×15×15cm，只能前进不得后退，禁履带/麦轮（2024 H 约束）。
- 起步对位技巧：车头绑激光笔辅助人工对准起始朝向，减少初始角度误差。
- 电源链路细节（本地缺）：12V 电池（输出 ≥2A）→ DC-DC 降 5V（大压差/大电流先用开关电源，发热小）→ LDO 降 3.3V 给主控。

2025 K 题（自动避障小车）新增要点：

- 场地为边长 2m 正方形，内有 4 白 5 黑共 9 根圆柱障碍，从入口 A 进、出口 C 出。
- 国一方案在 MSPM0G3507 官方开发板上实现多模式运动控制：基础避障通行、蛇形绕桩、绕柱旋转，并在此基础上做环境布局变化识别与自适应路径——比 2024 H 多了"动态避障 + 路径自适应"这一层。
- 测距避障常用 TOF 激光（如 VL53L0X，I2C，量程约 2m）替代或配合超声，单点 TOF + 转动可做简易扫描。整车复刻成本约 1000 元。

### 新增补充（2024-2025 获奖/开源复盘蒸馏）

来源见本节末"参考来源链接"。以下只补本地基础方案未覆盖的内容。

共性（多份方案一致，可当默认路线）：

- 普遍是"视觉处理器 + 下位机 MCU"双核分工：视觉端只算坐标/偏差，主控（STM32/MSPM0）只控执行（舵机/步进/夹爪）与显示，二者 UART 互联。追踪/瞄准/棋盘类用 OpenMV 一颗就够（可同时当视觉和主控）；单目测量要精度、要跑模型时才上算力更强的处理器。
- 任务切换走串口：按键在主控端选题号，主控通过 UART 把"题号/模式"发给视觉端，视觉端切换对应识别程序。
- 控制核心是"像素偏差 → PID 增量 → 执行机构"，闭环量普遍用增量式，避免一次性大角度过冲。
- 模块化先分调后合并：驱动、传感、视觉、显示、通信、激光各自单独调通再并入主程序，避免"牵一发动全身"。

**串口协议（本地只写"UART 坐标协议"，这里补具体做法）：**

- 多份复盘的事实标准：帧头 0x2C（部分再加第二帧头 0x12）、帧尾 0x5B，波特率 115200、8 位数据、无校验、1 停止位；OpenMV 端用 `ustruct.pack("<bbhhb", 0x2C,0x12,cx,cy,0x5B)` 打包发中心坐标。
- 关键决策——发偏差还是发坐标：发"像素偏差"（视觉端直接算"目标点 − 激光点/画面中心"的差）能把计算压在视觉端，主控只跑 PID，2023 E 国一即此法；发"绝对坐标"则由主控算偏差，灵活但占主控算力。瞄准/追踪建议发偏差。
- 实战坑：主控主频时钟源精度不足会误码/丢包；坐标先取整再发提稳；务必带帧头帧尾做校验防错位；视觉端 P4/P5 为 UART TX/RX，3.3V 电平，必须与主控共地。

**舵机 vs 闭环步进（本地只罗列两者，这里给取舍结论）：**

- 舵机：内置位置反馈、接线简单、功耗低，适合需要快速跟随的追踪/瞄准云台；缺点是高速性能差、瞬时电流大易抖、廉价舵机分辨率低。
- 步进：低速扭矩大、定位精度高、开环也能定位、无需反馈，适合要高精度或走固定路径（画圈/画字母/棋盘定点）的场合；缺点是要驱动器、速度慢。
- 结论：追踪/瞄准默认舵机云台，精度不够时先"换高精度数字/金属齿舵机 + 独立供电"，再考虑换步进；棋盘定点抓取类倾向 XYZ 步进龙门。

云台/执行机构实战处理（对应本地"抖动/回程误差/反光"风险给解法）：

- 舵机独立供电，与主控、视觉分电源，防瞬时大电流拉低电压导致复位。
- 适当降低舵机转速，避免支架/纸板形变与过冲抖动。
- 路径控制把线段细分成多小段逐步移动，让激光平滑走线。
- 三子棋执行机构取舍：XYZ 三轴步进龙门（类 3D 打印）比机械臂省掉逆运动学、回程误差更可控，是"最简满分"主流；机械臂自由度高但逆解与回程难调。抓取用电磁铁吸铁质棋子比夹爪更稳、回程误差更小。

差异 / 选型对比（按题型取舍）：

- 视觉端分档：OpenMV（生态成熟、可视觉+主控一体，追踪/瞄准/棋盘首选）→ K210/K230（算力强、可跑神经网络/YOLO，识别复杂目标或要精度时上）→ 树莓派 5（算力最强、多任务、接口丰富，单目测量等复杂算法首选）→ Jetson（更强但功耗大，一般无需）。
- 2025 C 单目测量国一用"树莓派 5 + STM32 双核 + YOLOv5-Lite + 电流检测电路"，实现距离 D、边长/直径 x、装置功耗的高精度实时测量。
- 单目测量标定与精度（本地无）：基于针孔/透视成像模型，先用已知尺寸标定物或棋盘格标定，求像素—实际比例与等效焦距；测距 D 由成像几何反解、边长 x 由像素长×比例换算。题目精度 D 误差≤5cm、x 误差≤1cm，并要监测 5V 供电电流 I_s（误差≤5%）。光照对边缘提取影响大，需固定/补光 + 自适应阈值。
- 2025 E 自瞄部分方案把"寻迹小车 + 云台瞄准"合一（MSPM0G3507 + K210/OpenMV + 8 路灰度 + MPU6050 + 编码器 + 舵机云台 + 激光），扩展任务要求激光沿指定路径画圈/字母。

说明：本节只补外部获奖报告/开源复盘中的可复用实现点，不重复上面的本地赛题索引。优先取 2024-2025；照度稳定台灯暂未找到更近且信息量足够的获奖复盘，保留 2021 国一复盘作为光照闭环模板。

1. 非接触控制盘：2025 电赛 I 题国一作品解析，基于光电与超声波的人机交互盘

   - 系统原理图/框图：4 路漫反射光电开关 S1-S4 + 1 路超声测距 S5 -> STM32H750XBH6 的 GPIO/定时器捕获/滤波与状态机 -> SPI DAC8830 输出目标电压 -> OPA541 功率放大 -> 520 直流风扇；TFT 串口屏显示距离、状态、电压和动作序列，WT588D 做语音提示。
   - 选型：主控 STM32H750XBH6；触发/手势为 4 路漫反射光电开关；测距为自研超声模块；执行为 DAC8830 + OPA541 + 直流风扇；显示/反馈为 TFT 串口屏、WT588D。按我们“全模块化”要求，可替换为 TCRT5000/CNY70 光电模块、HC-SR04/US-100 超声模块、IBT-2/L298N 风扇驱动模块和 OLED。
   - 解题思路：光电开关只做离散触发，按 Sx -> Sy 的触发顺序和时间窗识别启动、停止、正反转、调压等手势；超声距离 d 作为运行时间或目标电压的连续量；风扇电压用 DAC/功放或 PWM/驱动模块闭环。重点：超声具体频率和阈值整形源文未展开，比赛复用时按 40kHz 模块输出 ECHO 脉宽、定时器 1us 级捕获、多次中值/均值滤波；防误触用“触发顺序 + 最大手势时长 + 距离窗口 + 操作冷却时间”；模拟前端：源项目不是全现成模块，我们复用时必须模块化。

2. 超声信标定位：2025 电赛 J 题国二报告，超声波扇环定位系统

   - 系统原理图/框图：信标端 STM32F103C8T6/定时输出 -> L298N 双 H 桥增强驱动 -> 单个 40kHz 超声发射头 + LED 状态指示；定位端多路超声接收头 -> LF353 放大/滤波 -> STM32F103C8T6 定时/DWT 微秒计时 -> 扇区解算 -> OLED/蜂鸣器显示结果。
   - 选型：主控 STM32F103C8T6；发射驱动 L298N；接收前端 LF353 双 JFET 运放；超声为 40kHz 发射/接收换能器；执行/反馈为 OLED、蜂鸣器、LED。全模块化替代建议：40kHz 超声发射模块、带 LM393/运放整形的 40kHz 接收模块，或成品超声接收板 + MCU 捕获输入。
   - 解题思路：先用 TOF 得到距离，再用多接收点的到达时间差/路径差做三角定位和扇区判别。重点：源文给出声速温度修正 c = 331.4 + 0.6T，时间差用 DWT CYCCNT 转微秒，路径差用 Δd = c * Δt，发射频率稳定在 40kHz±1%；接收端 LF353 先放大，反馈小电容抑制高频噪声。防误触/抗多径：无信号显示 00，固定装置后多点重复测量，窄波束探头、增加接收探头数量、限定有效时间窗；模拟前端：源项目为自制 LF353 前端，不满足全模块化，复用时应改成成品接收/整形模块。

3. 声源定位/跟踪：2025 第十届立创电赛开源，基于 RA4L1 开发板的声源定位装置

   - 系统原理图/框图：6 个环形麦克风拾音单元/多组阵列 -> RA4L1-KB 的 SPI/ADC 同步采样 -> TDOA/GCC 时延估计 + 卡尔曼滤波 -> LED/小屏显示方向，或 GPIO/PWM 控制双轴云台和激光指向声源。
   - 选型：主控瑞萨 RA4L1-KB；麦克风阵列源文写 MP34DT01-M 环形阵列，另配 AD8221 前置放大和可选 ADS8684 16 位 ADC；执行为双轴舵机云台、激光指示、LED/显示屏；电源用 LM317/板载 LDO。全模块化建议：优先用成品麦克风阵列模块，或 MAX9814/INMP441 模块阵列 + 成品 ADC/声卡模块。
   - 解题思路：多麦克风先同步采样，取 256 点数据帧，加海明窗后 FFT，计算广义互相关/互功率谱，峰值位置作为到达时间差，再解算水平角/俯仰角；角度输出经卡尔曼滤波平滑后驱动云台。重点：多点到达时间差处理要保证同一定时基准，低频 20Hz 以下和高频 5kHz 以上先滤除；抗噪声/抗串音用独立腔体、隔音材料、屏蔽走线、模拟地和数字地磁珠单点连接、100nF 去耦、环形缓冲区防溢出；模拟前端：源项目混有自制前放，按我们要求应替换为成品麦克风/阵列/ADC模块。

4. 照度稳定调光：2021 电赛 K 题国一复盘，照度稳定可调 LED 台灯

   - 系统原理图/框图：4 路光敏电阻/照度采样 -> MSP430 ADC 均值滤波 -> PID/限幅 -> PWM -> LED 台灯驱动；按键切换手动/自动/手机 APP 模式，OLED/串口屏显示亮度，ESP8266 接收手机端调光值。
   - 选型：两块 MSP430；光照检测用光敏电阻阵列；显示用 OLED/串口屏；无线用 ESP8266；执行为 PWM 控制 LED 灯板。全模块化替代建议：BH1750/TSL2561/OPT3001 照度模块 + MOSFET 恒流/调光驱动模块 + OLED。
   - 解题思路：进入自动模式时先对 4 路光敏取 20 次均值作为目标基准，运行时每路再取 50 次均值；比较 4 路当前值与目标值差分，选最能代表外界扰动方向的一路参与闭环；PID 调 PWM，占空比限幅 0-999。重点：稳定区间内停止 PID 并清积分，避免闭环抖动；防误触/抗噪声靠多次均值、死区 HlightStable、输出限幅、模式状态机；模拟前端：源项目为光敏电阻直进 ADC，不是全现成模块，复用时改 BH1750/OPT3001 模块更稳。

5. 手势识别：2024 艾迈斯欧司朗竞赛开源，基于 dToF 的手势识别 2048 游戏机

   - 系统原理图/框图：TMF8821 dToF 多区域距离模块 -> I2C -> RP2040 Game Kit -> Edge Impulse/TinyML 手势分类 -> ST7789 屏幕和游戏/菜单控制。
   - 选型：主控 RP2040；传感器 TMF8821 dToF 模块，内部含 VCSEL、SPAD、TDC 和片上距离处理；显示 ST7789；软件为 Arduino 适配 + Edge Impulse。全模块化程度高，可直接把 TMF8821 或 VL53L5CX 这类多区域 ToF 模块接到 STM32/MSPM0。
   - 解题思路：dToF 输出多 zone 距离数据，板端按约 50Hz 采样并串口上传，每行特征用 TAB 分隔；采集 down、idle、right、left 四类样本，每类约 30 条，训练后导出 Arduino 推理库，板端 `run_classifier` 输出概率。重点：防误触用 idle 类、置信度阈值、连续多帧一致判定、低/高软件门限和 persistence 参数；抗环境光/多径主要依赖 TMF8821 片上降噪、去散射、置信度值和门限；模拟前端：全现成模块，符合我们模块化要求。

## 10. 比赛当天选题优先级
