# 2026浙江省赛_比赛背景与数据库任务说明

更新时间：2026-06-07

## 1. 比赛背景

目标比赛：2026 年 TI 杯第十四届浙江省大学生电子设计竞赛。

参赛状态：用户团队已经组队完成，目标为浙江专科/高职高专方向。

当前公开信息判断：

- 浙江高校已开始组织 2026 浙江省大学生电子设计竞赛报名、校内选拔和训练。
- 公开可见的高校通知显示，省赛要求倾向于 3 人组队、半封闭限时、实物作品测评。
- 校内选拔通知明确强调省赛推荐按实物组测评成绩排序，无实物作品不适合作为省赛主路线。
- 作品需要做到原理图、实物、源程序一致。

## 2. 我的任务定位

我的角色不是临时猜题工具，而是赛前外置资料库和方案推演系统。

赛前任务：

- 收集浙江省赛、全国高职高专题、培训网、历年题、公开项目、器件清单、规则要求。
- 把资料蒸馏成可直接调用的题型库、解法库、模块库、风险库。
- 建立“赛题关键词 -> 题型归类 -> 推荐方案 -> 器件/代码/测试流程”的快速匹配流程。
- 提前准备可复用模块：小车、测量仪器、非接触传感、视觉控制、声光交互、模拟前端。

比赛当天目标：

- 赛题发出后，快速判断最适合团队完成的题目。
- 第一时间给出稳拿基本分的最小可实现方案。
- 再给出冲高分方案和不建议路线。
- 帮助团队把 72 小时任务拆成可执行计划。

原则目标：

- 第一天完成可演示的基础系统。
- 第二天完善精度、稳定性和发挥项。
- 第三天集中测试、修正、写报告、固化演示流程。

## 3. 资料库需要覆盖的内容

### 官方与赛区资料

- 2026 浙江省大学生电子设计竞赛正式通知。
- 浙江赛区报名、组别、时间、器件、纪律、报告、测评规则。
- 浙江各高校公开的报名和选拔通知。
- 全国电赛与浙江省赛相关的实施过程说明。
- 2026 年 TI 杯/省赛相关器件与平台要求。

### 历年赛题资料

重点看：

- 2022、2024 浙江省赛题。
- 2019、2021、2023、2025 全国高职高专题。
- 近年与浙江省赛相似的本科低风险题，如测量仪器、视觉控制、小车、通信接收、声光交互。

### 技术资料

必须准备：

- STM32 / MSPM0 主控资料。
- ADC、PWM、定时器捕获、串口、I2C、SPI、OLED。
- PID、状态机、滤波、校准、误差分析。
- 运放、比较器、模拟前端、保护电路。
- 电机、舵机、步进、编码器、云台。
- 超声、TOF、红外、光电、麦克风、照度、摄像头。

### 开源项目资料

只提取：

- 架构思路。
- 驱动模板。
- 关键算法。
- 调试方法。
- 测试流程。

不直接照搬完整作品。

## 4. 当前最重要的押题方向

按浙江专科实战价值排序：

1. 测量仪器 / 信号检测 / 线路检测。
2. 小车 / 移动执行 / 任务控制。
3. 非接触感知 / 超声定位 / 光电控制。
4. 视觉识别 / 自动瞄准 / 二维平台。
5. 声光交互 / LED 显示 / 音频识别。
6. 模拟前端 / 运放滤波 / 功率或照度测量。

如果 2026 浙江省赛受模拟电子专题赛影响更强，则优先级调整为：

1. 模拟信号采集与测量。
2. 运放/比较器/滤波/ADC/DAC。
3. 传感器信号处理。
4. 小型控制装置。
5. 小车或机械执行作为备选。

## 5. 团队硬件策略

建议策略：

- 只画一块简单、稳定、通用的主控扩展板。
- 其他模块尽量使用现成模块。
- 主控板不做成某一道题的专用成品，保持通用性。

主控板必须覆盖：

- 主控最小系统。
- 下载调试口。
- 3.3V / 5V 供电。
- ADC 输入。
- PWM 输出。
- UART / I2C / SPI。
- 编码器接口。
- 舵机 / 电机控制信号口。
- OLED / 按键 / 蜂鸣器接口。
- 电源保护和清晰测试点。

## 6. 比赛当天决策流程

收到赛题后，按以下流程处理：

1. 先判定每道题的题型、硬件风险、机械风险、测评风险。
2. 排除高风险题：机械太复杂、功率太大、器件缺失、测试不可控。
3. 选择最适合团队现有模块库的题。
4. 先做 6 小时内可跑通的最小系统。
5. 再做基本要求全覆盖。
6. 最后冲发挥项。

输出格式固定为：

- 推荐选题。
- 不推荐选题。
- 基础得分方案。
- 高分冲刺方案。
- 器件清单。
- 硬件连接。
- 软件模块。
- 调试步骤。
- 测试表。
- 报告结构。
- 72 小时推进计划。

## 7. 合规边界

赛前可以充分准备资料、代码模板、模块库和训练方案。

比赛期间必须遵守当年浙江省赛正式规则。如果规则禁止与队外人员或外部智能系统交流，团队应使用赛前整理好的本地资料和模板，不把我作为实时外援。若规则允许查阅网络资料或工具，则也必须确保作品设计、制作、调试由参赛队员独立完成。

核心原则：用赛前准备提高速度，不用违规外援替代参赛队员完成比赛。

## 8. 最终目标

把所有资料变成一个能快速响应的工程数据库：

- 看到赛题，能马上判断做哪题。
- 看到指标，能马上拆出模块。
- 看到限制，能马上避开违规或高风险路线。
- 看到器件，能马上给出接线、代码和测试顺序。
- 第一天完成能演示的基础作品，后两天用于稳定和提分。
