# 03 · 规则与硬件限制核查

更新时间：2026-06-28

任务边界：以浙江省大学生电子设计竞赛为主，核查对硬件、主控、器件、外置算力板（Jetson / 树莓派 / OpenMV 等）的规则限制；全国电赛文件只作为浙江赛区选拔、后续国赛衔接和历年题面规律的参照。结论按“明确规则 / 题目个案 / 社区经验 / 不确定”分开。

## 0. 浙江省赛优先结论

如果只看你们当前最直接相关的浙江省赛，优先级是：

1. **浙江 2026 举办通知 + 附件 3 工作计划 + 浙江省赛章程**。
2. **浙江省赛当年题面和赛区答疑**。
3. 往年浙江题、全国题和全国规则，作为风险参考。

浙江省赛当前能确定的结论：

- **浙江省赛总规则未发现明文禁止 Jetson / 树莓派 / Linux SBC。**
- 浙江 2026 举办通知允许竞赛内容涉及嵌入式系统、可编程器件等方向，元器件或材料原则上由参赛学校提供。
- 浙江 2026 举办通知写明，赛题中有 **1-2 个题目** 的主要元器件指定采用 TI 单片机及相关模拟器件；这说明 TI 是部分题目的硬限制，不是所有题目的总强制条件。
- 浙江省赛最现实的硬件限制，仍然会出现在**单道题题面**：例如指定 MSPM0 / TI MCU、禁止摄像头、禁止 PC 机、禁止成品模块、要求自制某模块等。
- 浙江 2026 工作计划要求竞赛过程中拍半成品和成品照片，结束后密封实物作品和设计报告。Jetson / 树莓派若作为作品组成部分，应在作品中固定、随作品封存、脱机运行，不能像现场调试电脑一样临时接入。

一句话：**对浙江省赛而言，TX2 不是被总规则一票否决；真正决定能不能用的是当年浙江题面。**最稳的架构是“STM32 / MSPM0 做真实主控，TX2 只做视觉协处理器”，并准备一个不用 TX2 的降级方案。

## 1. 明确规则结论

### 1.1 总规则没有发现“统一禁止 Linux SBC / Jetson / 树莓派”的条款

**规则结论：**截至本次核查，本地官方章程、举办通知、实施过程说明、综合测评办法中，没有发现全国或浙江省赛总规则层面的“禁止 Jetson、树莓派、Linux 开发板、外置算力板”的统一条款，也没有发现总规则统一限定主控品牌或主控类型。

能确定的总规则口径是：

- 浙江省赛通知把竞赛内容定义为以电子电路和集成电路应用设计为基础，可涉及“嵌入式系统、可编程器件、射频及光电器件应用”等方向。
- 全国 / 浙江规则均强调：作品须由参赛队自主完成，竞赛期间不得与队外人员讨论交流，实物作品和设计报告是评审对象。
- 竞赛所需仪器设备、元器件、材料原则上由参赛学校提供。

这说明：**嵌入式系统、可编程器件本身在规则允许范围内；总规则不按“STM32 / TI / Jetson / 树莓派”这种品牌或板卡类型做一刀切限制。**

### 1.2 真正硬限制主要来自“单道题题面”

**规则结论：**硬件限制最常见、最有约束力的位置是题目 PDF 的“说明 / 要求 / 限制 / 不得 / 必须采用”条款。只要题面限定了主控、处理器、摄像头、测距方式、通信方式、成品模块或机械结构，就按题面执行。

因此，Jetson / 树莓派是否能用，不能只看总规则，必须逐题判断：

- 题面写“所有处理器必须使用 TI MCU”：Jetson / 树莓派不能作为处理器参与该作品。
- 题面写“某子系统必须由 MSPM0 控制”：该子系统不能交给 Jetson / 树莓派控制；其他未限定子系统是否可用，要看题面和答疑。
- 题面写“不得使用 PC 机（含笔记本、平板等）”：树莓派 / Jetson 是否被赛区视为 PC 类设备存在解释风险，必须看当年题目答疑。
- 题面只禁“台式计算机和笔记本电脑”：不等于自动允许所有 SBC，但风险低于直接禁 PC 机。

### 1.3 全国评审测试现场有“不得带入开发装置 / 个人计算机”等限制

**规则结论：**这一条主要来自全国电赛实施过程说明，对只参加浙江省赛的队伍不是第一优先规则；但浙江赛与全国赛规则体系高度相关，且浙江省赛本身也有作品拍照、封存和现场测试要求，所以它对 Jetson / 树莓派仍有参考价值。

全国 2025 实施过程说明明确要求，在赛区和全国评审测试中，除题目特殊要求外，参赛队的个人计算机、移动式存储介质、开发装置或仿真器等不得带入测试现场；实际制作实物中需软件编程的芯片必须事先下载，脱机工作。

对 Jetson / 树莓派的影响：

- 如果它只是临时调试电脑、开发板、外接算力盒、现场再接线的设备，风险很高。
- 如果它是封存作品的一部分，也仍需能脱机工作，不依赖外接电脑、U 盘、网络、键盘鼠标、显示器或现场重新部署程序。
- Jetson / 树莓派外形和用途接近“小电脑”，现场可能被要求说明其是否属于作品本体、是否违反题面“PC 机”限制。

稳妥做法：若确实使用 Jetson / 树莓派，应把它固定进作品结构，独立供电或受控供电，程序预先部署，开机自动运行，在报告中写清它的角色是视觉 / 推理协处理器，不是现场开发电脑。

### 1.4 全国综合测评比浙江省赛普通题更严，不能带 Jetson / 树莓派

**规则结论：**这一条不是浙江省赛普通作品制作阶段的规则，而是后续全国综合测评的规则。它不应反推为“浙江省赛普通题禁止 Jetson / 树莓派”，但如果你们后续进入国赛相关测评，需要按它准备。

全国综合测评实施办法采用封闭方式，测评所需开发板和元器件由全国组委会提供。参赛队可携带纸质资料、普通工具和少量无源器件，但不得携带电子材料、计算机、移动存储介质、开发装置、仿真器、单片机最小系统板、测试仪器、集成电路芯片等。

因此：**综合测评阶段不能指望带 Jetson、树莓派、OpenMV 或自备开发板入场。**这些板卡是否在前期竞赛作品中可用，不代表综合测评可带入。

### 1.5 TI 器件不是所有题目的强制要求，但会出现在指定题和专项奖里

**规则结论：**

- 浙江省 2026 举办通知写明，赛题中有 1-2 个题目的主要元器件指定采用 TI 单片机及相关模拟器件，供参赛学生选用。
- 全国 2025/2026 赛前说明均提到有题目指定 TI MSPM0 系列 MCU。
- TI 杯 / TI 专项奖按使用 TI 芯片情况及获奖情况评定。

所以：**TI 不是所有题目的总强制条件；但选择指定 TI 的题目时，题面的 TI 限制就是硬规则。**不用 TI 通常仍可参加其他非 TI 指定题，但拿不到相应 TI 专项奖。

### 1.6 “竞赛参考元器件及设备清单”不是总规则层面的唯一许可清单

**规则结论：**全国实施过程说明会在赛前公布“主要元器件和仪器设备清单”；历年文件名称常见为“仪器设备和主要元器件及器材清单”。清单的作用是提示可能用到的主要器件、设备和准备方向，不是覆盖题面限制的“唯一允许清单”。

判断顺序应为：

1. 题面硬限制优先。
2. 当年官方答疑 / 赛区通知优先于往年经验。
3. 元器件清单可作为准备参考，但不能拿来对抗题面“不得 / 必须 / 指定”的限制。

2025 全国清单中出现了“摄像头（可带处理器模块）”“嵌入式开发系统板”“微处理器最小系统板”等条目，说明官方准备范围内存在带处理器的视觉/嵌入式模块；但如果某一道题写了“不得使用 PC 机”“所有处理器必须使用 TI MCU”“不得安装摄像头”，仍按该题限制执行。

规律总结：

- **控制类小车 / 运动平台题**常见指定 TI MSPM0 或限制轮式结构、摄像头、外部导航、人工干预。
- **测量仪器题**常见限制主控、ADC、成品测量模块、专用仪器主板，防止用商品仪器拼装。
- **视觉题**常见限制摄像头数量、PC 机、非视觉测距方式、商品视觉测距模块。
- **通信 / 定位题**常见限制额外通信链路、外部定位方式、主动发射或外部导航设备。

## 3. 外置算力板合规性与测评坑

### 3.1 OpenMV / 带处理器摄像头模块

**规则层面：**2025 全国元器件清单列有“摄像头（可带处理器模块）”，这给 OpenMV、K210 摄像头模块、带处理器视觉模块一类设备留下了官方准备依据。

**但限制边界：**

- 题面若要求“不得安装摄像头”，不能用。
- 题面若要求“所有处理器必须 TI MCU”，不能用。
- 题面若要求“不得利用视觉测距类商品改装”，不能把成熟视觉测距商品模块当黑盒使用。
- 题面若只允许一个摄像头，OpenMV 也只能按一个视觉输入算，不能再外接第二路视觉规避。

### 3.2 树莓派 / Jetson

**规则层面：**没有找到总规则明文禁止树莓派或 Jetson。它们可被理解为嵌入式系统 / 外置协处理器，但不代表每道题都允许。

**风险层面：**树莓派 / Jetson 跑通用 Linux，形态接近小型计算机，遇到“不得使用 PC 机”“不得带入个人计算机 / 开发装置”的表述时，解释风险显著高于普通 MCU 板或 OpenMV。

建议判断：

- 题目指定“TI MCU / MSPM0 控制”：Jetson / 树莓派不能替代指定控制器。
- 题目限定“所有处理器必须 TI MCU”：Jetson / 树莓派不能用。
- 题目禁“PC 机（含笔记本、平板等）”：不要默认树莓派 / Jetson 安全，必须查题目答疑或问赛区。
- 题目只禁“台式计算机和笔记本电脑”：可用空间更大，但仍要能说明它是作品嵌入式部件而不是现场电脑。
- 题目允许视觉，且元器件清单包含“摄像头（可带处理器模块）”：OpenMV 类通常比 Jetson / 树莓派更低风险。

### 3.3 测评常见坑

**现场演示：**作品应能上电后独立运行。不要依赖现场 SSH、远程桌面、网络下载、手动运行脚本、外接显示器键鼠调参。

**作品封存：**浙江 2026 工作计划要求竞赛期间拍半成品 / 成品照片，竞赛结束后负责人和巡视员共同密封实物作品和设计报告。Jetson / 树莓派若属于作品本体，应在半成品和成品照片中清楚出现，并随作品一起封存。

**程序代码：**全国 2025 实施过程说明主要提交设计报告电子文档和实物作品；浙江 2025 工作计划曾要求提交最终版程序代码（可编程器件与 MCU）。如果使用 Jetson / 树莓派，建议主动把其启动脚本、推理程序、模型版本、依赖说明写入报告或附录，避免“只交 MCU 代码而视觉端不可复现”的问题。

**脱机工作：**全国评审测试要求需软件编程的芯片事先下载、脱机工作。Jetson / 树莓派应预装系统和程序，禁止测试现场再用 U 盘、手机热点、笔记本或仿真器补部署。

**供电和安全：**Jetson TX2 这类板卡功耗和供电要求高于 MCU。若作品电源、开关、急停、接地、线缆固定做不好，测试现场更容易被视为作品不稳定，而不是规则问题。

## 4. 不确定 / 未找到

- **未找到**全国或浙江总规则中“禁止 Jetson / 树莓派 / Linux SBC”的明文条款。
- **未找到**总规则统一规定“必须使用某品牌 MCU”或“所有题都必须使用 TI”的条款。
- **未找到**2026 年全国普通赛题的具体题面；2026 具体能否使用 Jetson / 树莓派，仍要等当年题面和答疑。
- **未找到**2026 年浙江省赛“主要元器件和仪器设备清单”的本地文件；本地已有 2026 浙江举办通知和工作计划。
- **不确定**当题面写“不得使用 PC 机”但未列举树莓派 / Jetson 时，各赛区是否会把 Linux SBC 解释为 PC。这个问题必须以当年题目答疑、赛区测试组口径为准。
- **社区经验**显示，树莓派、Jetson、OpenMV 是否允许在不同年份、不同题目、不同赛区有过讨论和争议；这些讨论只能作为风险提示，不能当规则引用。

## 6. 一句话给当前 Jetson TX2 架构

如果当前方案是“STM32 / MSPM0 做真实主控，Jetson TX2 只做视觉协处理器”，在**总规则层面未发现直接违规**；但一旦选择的题目出现“所有处理器必须 TI MCU”“不得使用 PC 机”“不得安装摄像头”“某控制任务必须由 MSPM0 完成”等条款，就必须按题面收缩架构。赛前准备应保留纯 MCU / OpenMV 低风险替代方案，并在题目公布后逐条复核。
