# 2026 浙江省赛_赛前执行清单与训练排期

更新时间：2026-06-07

用途：把已有的题型预测、方案分组、GitHub 解法雷达压缩成赛前可执行清单。重点不是继续收集资料，而是让团队在赛题发布前把硬件、代码模板、调试流程和选题决策表准备到能直接使用的状态。

适用队伍：浙江省赛专科/高职高专方向，目标是优先稳拿基础分，再争取发挥分。

## 1. 当前主线判断

最值得押注的三条主线：

1. 测量仪器 / 信号检测 / 简易参数估计。
2. 自动小车 / 运动控制 / 执行机构任务。
3. 视觉识别 / 非接触感知 / 简单瞄准或定位。

备选主线：

1. 超声 / 声光定位 / 距离测量。
2. 音频识别 / 简易通信 / 接收机。
3. 运放滤波 / 小功率功率测量。

不建议作为默认主攻：

1. 高功率电源、并联变换器、有源滤波、能量回馈。
2. 无人机、复杂机械臂、复杂三维结构。
3. 需要现场重新学习新平台或大规模移植的工程。

## 2. 必买硬件清单

### 主控与调试

- MSPM0G3507 开发板或最小系统板：至少 2 块。
- STM32F103/F407 熟悉板：至少 1-2 块，作为备选和快速验证。
- ST-Link / XDS110 / DAP-Link：至少 2 个。
- USB-TTL：至少 3 个。
- OLED 0.96 或 TFT 小屏：至少 3 块。
- 按键、旋钮、蜂鸣器、LED、拨码开关若干。

### 测量与模拟前端

- 常用运放模块或芯片：LM358/TLV 系列/轨到轨低压运放各若干。
- 比较器模块或芯片：LM393/TLV 系列若干。
- 基准源、可调电位器、精密电阻、电容、电感包。
- 分压保护小板：限流电阻、钳位二极管、TVS、保险丝。
- 电压/电流采样模块：霍尔电流、采样电阻放大模块各备一套。
- 可调信号源或函数发生器；没有则准备 DDS/AD9833 模块作低端替代。

### 小车与执行

- 编码器电机小车底盘：至少 1 套完整可跑。
- TB6612 / DRV8833 / 合适电机驱动：至少 3 块。
- 8 路灰度阵列、红外避障、超声模块、ToF 模块各若干。
- IMU：JY901S、MPU6050 或 LSM6DSV16X 至少 1-2 个。
- 舵机、步进电机、A4988/DRV8825、二自由度云台。

### 视觉与感知

- OpenMV 或 MaixCam：至少 1 个熟悉平台。
- USB 摄像头 + 可运行 OpenCV 的小电脑/开发板：作为备选。
- 激光笔、色块、棋盘格标定板、固定光源。

### 电源与安全

- 可调直流电源：至少 1 台。
- 5V/3.3V 稳压模块、锂电池、保险丝、总开关、接插件。
- 所有电机/舵机/云台单独供电，和主控共地。

## 3. 赛前软件模板

每个模板必须能单独跑通，并有串口调试输出。

### 主控基础模板

- GPIO 输入输出。
- UART 收发和 printf 日志。
- I2C OLED 显示。
- SPI 备用。
- 定时器 PWM。
- 定时器输入捕获。
- ADC + DMA 固定采样。
- Flash/EEPROM 参数保存。

验收标准：

- 上电后显示版本号、模式、关键传感器读数。
- 串口能输出原始数据和计算结果。
- 任何模块异常时有状态码，而不是静默失败。

### 测量模板

- ADC DMA 采样 1024 点。
- 峰值、均值、RMS、频率基础计算。
- 定时器输入捕获测频。
- 简单 FFT 或 Goertzel 备用。
- 校准参数：零点、比例系数、满量程。
- OLED 显示测量值、量程、误差状态。

第一天可交付目标：

- 单量程可测、可显示、可校准。
- 能用标准信号源或已知元件验证。

### 小车模板

- 电机 PWM 输出。
- 编码器测速。
- 速度 PID。
- 基础动作：直行、转向、定距停车、急停。
- 灰度循迹或 ToF/超声避障。
- 任务状态机和超时保护。

第一天可交付目标：

- 小车能稳定直行、转弯、停车。
- 不依赖纯延时完成核心动作。
- 所有传感器有原始值显示或串口输出。

### 视觉/云台模板

- OpenMV/MaixCam 输出目标坐标。
- MCU UART 接收坐标。
- 舵机或步进云台按误差移动。
- 限幅、死区、低通滤波。
- 手动模式和自动模式切换。

第一天可交付目标：

- 摄像头能识别目标并输出坐标。
- 执行机构能移动到指定点附近。
- 不先追求复杂识别模型。

## 4. 两周滚动训练排期

### 第 1-2 天：主控和显示

目标：

- 建好一个可复用主控工程。
- OLED、串口、按键、蜂鸣器全部跑通。
- 建立统一日志格式：时间、模式、原始值、计算值、状态码。

产物：

- `base_mcu_template`。
- 一页接线表。
- 一张常用 IO 分配表。

### 第 3-4 天：ADC 和测量

目标：

- ADC DMA 固定采样。
- 测峰值、RMS、频率。
- 用函数发生器或 DDS 做 10Hz、100Hz、1kHz、10kHz 验证。

产物：

- `measure_adc_template`。
- 校准表模板。
- 输入保护电路草图。

### 第 5-6 天：小车闭环

目标：

- 编码器测速。
- 速度 PID。
- 定距直行、原地转向、急停。
- 灰度或 ToF 接入。

产物：

- `car_control_template`。
- PID 参数记录表。
- 小车动作状态机表。

### 第 7 天：复盘和封装

目标：

- 把前三类模板整理成离线包。
- 每个模板保留最小可运行版本。
- 不把临时代码混进正式模板。

产物：

- 离线工程包。
- 模块验收清单。
- 现场调试步骤卡片。

### 第 8-9 天：视觉或非接触感知

目标：

- OpenMV/MaixCam 识别色块、圆点、矩形或棋盘格。
- UART 输出坐标。
- MCU 接收并控制舵机/云台。

产物：

- `vision_uart_template`。
- 摄像头固定支架方案。
- 光照和阈值记录表。

### 第 10-11 天：模拟前端和保护

目标：

- 做一个通用输入保护小板。
- 验证分压、钳位、运放缓冲、低通滤波。
- 记录可接入 ADC 的安全范围。

产物：

- 前端接线图。
- 保护电路 BOM。
- 示波器波形照片。

### 第 12-13 天：模拟赛

目标：

- 从历年题中抽一题，限制 8 小时完成最小系统。
- 只允许使用已准备模板和模块。
- 记录卡住的位置。

推荐题：

- 2021 J 周期信号波形识别及参数测量装置。
- 2023 J 线路故障自动检测系统。
- 2024 H 自动行驶小车。
- 2025 K 自动避障小车。
- 2025 E 简易自行瞄准装置。

### 第 14 天：封版

目标：

- 冻结可用工程。
- 整理工具箱、模块、线材、备用板。
- 每个模块贴标签并写清电压、接口、注意事项。

产物：

- 赛前离线资料包。
- 硬件箱清单。
- 当天选题评分表。

## 5. 比赛当天选题评分表

每道题按 0-5 分打分，总分越高越优先。

| 维度 | 评分说明 |
|---|---|
| 题型匹配 | 是否命中测量、小车、视觉/感知三条主线 |
| 硬件现成度 | 现有模块是否能直接搭起来 |
| 第一日可完成度 | 6-10 小时内是否能做出可演示基础系统 |
| 测试可控性 | 能否用现有仪器验证指标 |
| 机械风险 | 是否需要临时加工复杂结构 |
| 模拟前端风险 | 是否可能烧板、漂移、噪声不可控 |
| 代码复用度 | 是否能套用已有模板 |
| 发挥分空间 | 基础功能稳定后是否有明确加分路线 |

优先选择：

- 总分高于 30。
- 第一日可完成度不低于 4。
- 硬件现成度不低于 4。
- 机械风险和模拟前端风险至少有一个低。

直接降级或放弃：

- 第一日可完成度低于 3。
- 需要临时学习新主控或新开发环境。
- 需要高功率、高压、大电流调试且没有完整安全条件。
- 基础要求依赖复杂机械加工或无人机飞行稳定性。

## 6. 72 小时推进模板

### 0-2 小时：读题与选题

- 所有人独立读题，圈出输入、输出、指标、限制、评分项。
- 用评分表给每题打分。
- 选一个主攻题，一个备用题。
- 明确第一天最小系统。

### 2-6 小时：最小系统

- 硬件只搭核心链路。
- 软件只跑最基本功能。
- 每个传感器和执行器先单独验收。
- 串口和屏幕必须能显示关键状态。

### 6-12 小时：基础要求覆盖

- 逐条覆盖基本要求。
- 建立测试表。
- 任何不稳定功能先降级，不影响主链路。

### 12-24 小时：完整演示

- 做出可重复演示流程。
- 拍照记录硬件、波形、测试数据。
- 固定接线和机械结构。

### 24-48 小时：精度和稳定性

- 做校准。
- 调 PID 或阈值。
- 增加异常恢复和手动兜底模式。
- 整理报告框架。

### 48-72 小时：发挥项和报告

- 只加不会破坏基础功能的发挥项。
- 完成测试数据、误差分析、照片和框图。
- 固化演示步骤。
- 最后 4 小时不再大改核心代码和硬件。

## 7. 团队分工

### 硬件/机械

- 负责供电、接线、传感器安装、执行机构固定。
- 维护硬件清单和故障替换方案。
- 所有接口贴标签，标明电压和方向。

### 嵌入式/控制

- 负责主控工程、驱动、状态机、PID、显示和串口。
- 保证每个模块有独立测试模式。
- 不在比赛当天重构工程。

### 测试/报告

- 负责读题拆指标、测试表、误差表、照片、报告框架。
- 每完成一个功能就记录验证数据。
- 维护演示流程，保证评测时能复现。
