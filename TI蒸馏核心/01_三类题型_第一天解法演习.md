> 配合 `00_比赛当天_读我_作战总纲.md` 使用。这是按总纲七步协议,对三类押注题型预跑的"第一天最优可行解法"。

## 一、小车控制类

> 典型"自动行驶/避障小车"题。

**① 指标拆解(评分点,当天填真值)**
循迹是否到位 / 定距停车误差(cm) / 转向角度精度 / 避障是否触发 / 完成时间(s) / 是否要求路径规划。

**② 题型命中 / TX2 上不上**
归"方案组B 小车"。纯巡线/定距/避障 → **只用 MCU,TX2 不上**;要视觉识别目标/颜色才让 TX2 上(发坐标给 MCU)。**第一天默认不上 TX2**,先把车跑稳。

**③ 选型清单(全模块)**
| 角色 | 型号/模块 | 状态 |
|---|---|---|
| 主控 | MSPM0G3507 核心板 | 已在清单 |
| 电机驱动 | TB6612FNG ×2 | 已在清单 |
| 底盘 | 编码器电机两轮差速 | 已在清单 |
| 姿态 | JY901S(航向角) | 已在清单 |
| 循迹 | 8路灰度模块 | 已在清单 |
| 避障/定距 | VL53L1X 或 HC-SR04 | 已在清单 |
| 显示 | 0.96 OLED | 已在清单 |

**④ 第一天最小成品**
> 车能**直行、转弯、定距停车**,编码器速度闭环稳定,按键手动选模式。

逐环验证:电机正反转调速 → 编码器读到速度 → 速度 PID 闭环不抖 → 定距停准 → JY901S 角度闭环转向准。

**⑤ 第二天加分项**
灰度循迹 PID、避障阈值、任务状态机、补偿表、(需要才上)TX2 视觉。

**⑥ 可抄资源**
- `abcuer_2024-NUEDC-H-TI_CAR`:`Control\pid.c / track.c / angle.c / dist.c`、`Hardware\encoder.c / motor.c / gray.c / jy901s.c` —— 轻量模板,直接移植 MSPM0。
- `JamieK32_ti-contest-2025`:完整工程,拆驱动/状态机/VL53L1X。

**⑦ 已知坑**
轮胎打滑/电源压降、只靠延时控制、传感器装不稳、地面光照变化导致循迹失效。

**⑧ 补充(2024-2025 复盘)**
- **机械硬约束(2024H,采购按此)**:车体 ≤25×15×15cm、只准前进禁后退、禁履带/麦轮 → 底盘买**两轮差速编码器**,卡尺寸。
- **控制**:环路**只用 PD,去掉 I**;**双环状态机** = 直道陀螺仪航向 PD / 弯道灰度 PD 切换;角度偏差做 **±180° 跨零处理**。
- **电源链路**:12V电池 → DC-DC 降 5V → LDO 降 3.3V,每条供电链单独走。
- **选型微调**:IMU 预算够可升 HWT101(零漂更小);ToF 可用 VL53L0X/L1X。

## 二、视觉识别 / 自瞄类

> 典型"运动目标追踪 / 自行瞄准 / 单目测量"题。

**① 指标拆解(评分点)**
能否识别目标(色块/形状/激光点)/ 定位坐标误差(像素或cm)/ 瞄准命中精度 / 追踪响应时间 / 是否要求测距或尺寸测量。

**② 题型命中 / TX2 上不上**
归"方案组C 视觉"。**这是 TX2 唯一的主场**:TX2(USB摄像头+OpenCV)只管识别、输出坐标;MCU 只管收坐标、驱动云台/舵机执行。串口帧 `0xAA LEN DATA 0x55` @115200。

**③ 选型清单(全模块)**
| 角色 | 型号/模块 | 状态 |
|---|---|---|
| 视觉端 | TX2 + USB摄像头(免驱UVC) | TX2已有,摄像头待购 |
| 执行端主控 | MSPM0G3507 / STM32 | 已在清单 |
| 瞄准执行 | 二维云台 + **闭环步进**(优先)或 MG996R舵机 | 已在清单 |
| 指示 | 激光笔模块 | 已在清单 |
| 显示 | 0.96 OLED | 已在清单 |

> **选型修正(2024-2025 复盘,推翻"一律优先步进")**:**追踪/自瞄默认用高精度数字舵机**——内置反馈、接线简单、响应快,够用;抖就独立供电+降速+路径细分。**只有棋盘定点 / 走固定路径才上闭环步进**;棋盘抓取用 XYZ 步进龙门 + 电磁铁(比机械臂/夹爪更稳、回程更可控)。

**④ 第一天最小成品**
> 摄像头**识别到目标 → 串口把坐标发给 MCU → 云台/舵机移动到指定点**。

逐环验证:TX2 摄像头出画面(USB,`VideoCapture`)→ OpenCV 找到目标质心(x,y)→ 串口发出 → MCU 收到坐标 → 执行机构动到该点。**先能动,不追识别完美。**

**⑤ 第二天加分项**
目标居中 PID、前馈角度+滤波(卡尔曼/滑动)、自动决策(三子棋走法/追踪)、精度标定补偿。

**⑥ 可抄资源**
- `DBinK_2023-NUEDC-E-OpenCV`:`quad_detector.py / point_detector.py / stream.py` —— 四边形/中心点/红绿光点识别(Python,匹配你们路线)。
- `STM32xxx_Electromagnetic-gun`:OpenMV+舵机/云台瞄准思路。
- `JamieK32_ti-contest-2025`:`maixcam_project\uart.py` + `maix_cam.c` —— 现成串口坐标协议。

**⑦ 已知坑**
机械结构最耗时(别做复杂机构)、光照/反光、坐标标定不准、执行抖动与回程误差;CSI 板载摄像头别用,上 USB 摄像头。

**⑧ 补充(2024-2025 复盘)**
- **串口:追踪/自瞄"发偏差不发坐标"** —— 视觉端直接算目标相对中心的偏差,主控只跑 PID(2023E 国一即此法);单目测量/棋盘才发坐标。帧要**帧头帧尾校验 + 共地**,防主频精度不足丢包、坐标取整。
- **执行选型见上方修正**:追踪/自瞄默认数字舵机,棋盘/固定路径才步进。
- **标定**:单目测量先做针孔/透视标定求"像素↔实际"比例;光照固定 + 补光 + 自适应阈值。
- **视觉端档位参考**:OpenMV→K210/K230→树莓派5→Jetson;2025C 国一用树莓派5+STM32+YOLOv5-Lite。你们的 **TX2 属最重档,算力够,瓶颈在跑通不在性能**。

## 三、声光 / 传感类

> 典型"非接触控制 / 超声定位 / 声源定位 / 照度调光"题。

**① 指标拆解(评分点)**
测距/定位精度(cm)、响应时间、控制量误差(如电压≤0.1V)、识别正确率、稳定可复现度。

**② 题型命中 / TX2 上不上**
归"方案组D 非接触感知"。**绝大多数纯 MCU,TX2 不上**(2025 I 题就是纯 STM32)。只有"视觉辅助定位"才让 TX2 出场。

**③ 选型清单(全模块)**
| 角色 | 型号/模块 | 状态 |
|---|---|---|
| 主控 | STM32F103 / MSPM0G3507 | 已在清单 |
| 测距 | 超声模块(CS100A/HC-SR04)或 VL53L1X ToF | 已在清单 |
| 手势/触发 | 光电开关模块(漫反射)×4 | 需按题补 |
| 声源 | 麦克风模块 MAX9814 ×多个 | 已在清单 |
| 执行 | L298N/TB6612 + 风扇/电机 / 大功率LED / 继电器 | 已在清单 |
| 显示 | 0.96 OLED | 已在清单 |

**④ 第一天最小成品**
> **单通道传感器可靠读数 → OLED 显示距离/状态 → 驱动一个输出动作**(如 PWM 调风扇/LED)。

逐环验证(参 I题):超声测出距离 d(`d=脉宽×340/2`)→ OLED 显示且手移动数值跟着变 → 一路触发信号进 GPIO → PWM 出占空比驱动执行 → 串起来。

**⑤ 第二天加分项**
多点定位(TDOA)、抗干扰/防误触(I题:脉宽>1200μs不采)、闭环精度(电压误差≤0.1V)、自动校准。

**⑥ 可抄资源**
- **2025 I题报告**:STM32+CS100A超声+光电隔离+L298N风扇+OLED 全链,选型/计算/测试表可直接套。
- `framist_NUEDC2022-E`:声源定位/TDOA/信号调理(算法可借,工程偏老)。

**⑦ 已知坑**
超声多径与环境噪声、接收阈值不稳、传感器安装角度不一致、闭环无死区导致抖动;模拟前端一律用现成模块,别裸接 MCU 的 ADC。

**⑧ 补充(2024-2025 复盘)**
- **最契合"全模块化"的路子**:手势/非接触距离题**优先用多区域 dToF 模块(TMF8821 / VL53L5CX)**,I2C 直连 MCU,片上降噪 + 多 zone 距离,比自制超声+光电干净得多(2024 dToF 手势开源即此法)。
- **超声计时/精度**:声速做温度修正 `c = 331.4 + 0.6T`(m/s);用定时器 1µs 级捕获或 DWT 计时,多次中值/均值滤波。
- **防误触升级**(2025 I 题国一):用"**触发顺序 + 最大手势时长 + 距离窗口 + 操作冷却时间**",比单一脉宽阈值稳。
- **声源定位**(立创 RA4L1):多麦同步采 256 点 → 海明窗 FFT → GCC 广义互相关求 TDOA → 卡尔曼平滑驱动云台;滤除 <20Hz/>5kHz;模拟地数字地磁珠单点共地 + 100nF 去耦。
- **照度闭环**(2021 K 题):多路照度取均值作基准,进稳定区**清积分、停 PID** 防抖,PWM 限幅。
- **全模块替代对照**(满足全模块化):光电→TCRT5000/CNY70;超声→HC-SR04/US-100;麦克风→MAX9814/INMP441;照度→BH1750/OPT3001;风扇驱动→IBT-2/L298N 模块。
