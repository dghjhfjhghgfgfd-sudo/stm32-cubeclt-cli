## 一、我们是谁 / 打什么

- **赛事**:浙江省大学生电子设计竞赛(省赛,A–H 多题并行)。
- **队伍定位**:专科队,主攻**软件含量高的三类题** —— 小车控制 / 视觉识别·自瞄 / 声光·传感。
- **已有板子**:NVIDIA Jetson TX2(视觉端,已有)。执行端单片机用 **MSPM0G3507 或 STM32**。
- **硬性偏好**:**能用模块就全用模块**,不自己焊分立电路(运放/比较器/麦克风等一律用现成模块板)。

## 二、固定架构(不管哪题,先往这上面套)

```
传感器/摄像头 → 算(TX2 或 MCU) → 串口/信号 → 执行(电机/舵机/声光) + OLED 显示
```
- 视觉题:TX2(USB摄像头+OpenCV) ──UART──→ MCU 执行。串口帧 `0xAA LEN DATA 0x55` @115200。
- 纯传感/声光/小车题:常常**只用 MCU**,TX2 不一定上场(例:2025 I 题非接触控制盘就是纯 STM32)。
- 先把这条骨架的**最小版打通**,再加功能。

### TX2 用不用 · 看题面判定(规则核查见 `03_规则与硬件限制_核查.md`)

- 省赛**总规则不禁** TX2 / Jetson / Linux SBC;真正限制只在**题面**。
- 题面出现以下任一 → **TX2 退场,走纯单片机**:"必须 TI MSPM0 / 所有处理器须用 TI MCU / 不得使用 PC 机 / 不得安装摄像头"。
- 否则 → **TX2 可用**,但必须作为**作品本体、开机自启、脱机运行**(systemd 已做到);**不能当现场开发电脑**——测评禁带个人电脑/开发装置/仿真器。
- 若晋级**国赛综合测评**:更严,自备 Jetson/树莓派/开发板不能带——届时另算,不影响省赛。
- 结论:不管哪种,**STM32/MSPM0 永远是真主控**;TX2 只是"能用就加视觉、被禁就拿掉"的协处理器。所以两条路线都要能打。

## 三、三条核心打法(从历年国奖报告提炼,最重要)

**1. 指标驱动 —— 分是"达标 + 可复现"换来的,不是功能多。**
拿到题第一件事:抠死题目的**关键量化指标**(精度/误差/时间/距离)。所有动作朝"测得到、达标、能重复"走。先别堆功能。

**2. 最简选型 —— 在成熟方案里选够用、稳定、好调的那个。**
国奖报告的"方案论证"本质就是二选一选简单可靠的(STM32 不选 Arduino、PWM 不选 DAC、L298N 不选自搭H桥)。**够用 + 稳定 + 好调 > 复杂炫技**。这也是"全用模块"的理由。

**3. 固定骨架 + 固定报告模板 —— 比赛时先套框架,再填内容。**
系统骨架见上;报告永远是五段:**方案论证 → 理论分析与计算 → 电路与程序设计 → 测试方案与结果 → 结论**。我们出真实测试数据,AI 套模板写报告(报告是一半的分)。

## 四、第一天必须出成品(铁律)

- 第一天目标 = **一个能现场测分的最小完整系统**,不是完整功能。
- 骨架任何一环没通,后面全白搭 —— 按"传感器读到数 → 算出结果 → 执行动起来 → OLED 显示"逐环验证。
- 高级功能、精度优化、自动量程、决策逻辑 —— **都放第二天**。

## 五、给 Claude 的产出协议(拿到题目后这样回我)

读完本总纲 + 题目后,按以下结构给出解法:

1. **指标拆解**:把题目的关键量化指标逐条列出(这是评分点)。
2. **题型命中**:归到三类骨架的哪一类,TX2 上不上场。
3. **选型清单(全模块)**:主控 + 传感器模块 + 执行模块 + 显示,给具体型号,标"我们清单里已有 / 需现买"。
4. **第一天最小成品**:一句话定义"能测分的最小系统",再给逐环验证步骤。
5. **第二天加分项**:达标后再上的功能/精度优化。
6. **可复用代码/资源**:指向 `GitHub资料库` 里能直接抄的工程和模块。
7. **已知坑**:从报告/经验提炼的现场踩坑点。

原则:**最优 = 第一天能做出达标成品的方案**,不是理论最强的方案。冲突时给 A/B 路线。
